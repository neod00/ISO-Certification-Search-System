💡 Idea, Background & History

The Idea

When LRQA sales team members propose ISO certifications to potential clients, they must first research what ISO certifications the company already holds. This manual process requires visiting multiple websites (Google News, Naver News, company websites, official certification databases) and reading through numerous news articles to extract certification body information—taking 10-15 minutes per company. What if we could automate this entire research process using AI?

Our solution combines ChatGPT (GPT-4o-mini) natural language processing with multi-source web scraping to create an intelligent research platform that automatically collects, analyzes, and extracts ISO certification information for potential clients in seconds—enabling sales teams to prepare targeted proposals faster and reducing research time by 70% with 85% accuracy in certification body extraction.

💭 Background

LRQA sales team members need to quickly research potential clients' existing ISO certifications before proposing new certifications. This information is scattered across multiple sources: official databases (KCN), news articles, company websites, and blogs. Manually extracting certification body information from news article bodies requires reading and understanding context, making it time-consuming and error-prone for sales professionals.

🔬 Research & Development

Development started in Sep 2025 with React + TypeScript frontend, tRPC + Node.js backend, and GPT-4o-mini for news analysis. We implemented a hybrid search approach that provides immediate results while continuously improving accuracy through background GPT analysis.

Multi-Source Web Scraping

People: Backend engineers
Process: Parallel scraping from 5+ sources:
• Google News (Korean & English)
• Naver News
• Naver Blog
• Company Websites
System: Axios + Cheerio, timeout management, error recovery

Hybrid Search Architecture

People: Full-stack developers
Process: Immediate response (3-5s) from cache + DB + basic scraping, then background GPT analysis to improve cache for next search
System: PostgreSQL (Supabase), 24-hour caching, async background processing

Automatic Language Detection

People: Frontend developers
Process: Detects Korean/English characters in company name, adjusts search terms and news sources accordingly
System: Regex-based detection, language-aware scraping, multilingual GPT prompts

⚖️ Risks & Mitigation Strategies

🔒 Data Security & Privacy

Risk: Sensitive company information exposed during scraping and API calls

✅ Mitigation:
  • HTTPS/SSL encryption for all data transmission
  • Environment variables for API keys (not hardcoded)
  • Input validation and sanitization
  • Only public information is collected (no authentication required)
  • CORS policies to restrict unauthorized access

🤖 AI Accuracy & Reliability

Risk: GPT-4o-mini may extract incorrect certification information from news articles

✅ Mitigation:
  • Source attribution for all extracted information
  • Human review capability (users can verify sources)
  • Multiple news articles analyzed (top 3) for cross-validation
  • Structured JSON output with validation
  • Fallback to basic scraping if GPT analysis fails

🔗 Web Scraping Reliability

Risk: Target websites may change structure, blocking scraping or returning errors

✅ Mitigation:
  • Robust error handling with graceful degradation
  • Multiple data sources (if one fails, others continue)
  • Timeout management (4-8 seconds) to prevent hanging
  • Database cache as fallback (24-hour validity)
  • Regular monitoring and selector updates

💰 API Cost Overruns

Risk: Unexpected high usage could lead to significant OpenAI API costs

✅ Mitigation:
  • GPT-4o-mini chosen for cost efficiency (~$0.001/search)
  • Background analysis only (not blocking user requests)
  • Maximum 3 news articles analyzed per search
  • 24-hour caching reduces redundant API calls
  • Usage monitoring and alerts

⚡ Serverless Function Timeouts

Risk: Netlify free plan has 10-second function timeout; complex searches may exceed this

✅ Mitigation:
  • Hybrid search: immediate response (3-5s) + background analysis
  • Scraping timeout set to 4 seconds (well under limit)
  • Parallel processing where possible
  • Cache-first strategy reduces processing time
  • Upgrade path to Pro plan if needed (60s timeout)

🌍 Scalability Concerns

Risk: High concurrent usage may strain serverless functions and database

✅ Mitigation:
  • Connection pooling for PostgreSQL (Supabase)
  • 24-hour caching reduces database queries
  • Serverless auto-scaling handles traffic spikes
  • CDN for static assets (Netlify)
  • Database indexing for fast searches

⚠️ Limitations & Known Issues

Early Version Constraints

This platform is currently in its initial version (v0.1) and is under continuous development. As such, users may encounter certain limitations:

📊 Incomplete Search Results

Issue: Some company searches may return no results or incomplete information if the company's ISO certification data is not available in the public sources we scrape.

💡 Current Status:
  • This occurs when information is not publicly reported in news articles or official databases
  • Coverage varies by company size and public disclosure practices
  • We are continuously expanding our data sources to improve coverage

❌ Incorrect or Missing Information

Issue: Due to the AI-powered nature of information extraction, some data may be incorrectly parsed from news articles, or certain fields (certification bodies, dates) may be missing.

💡 Current Status:
  • AI accuracy is approximately 85% but may vary by source quality
  • All results include source attribution for manual verification
  • We recommend verifying critical information using source links

🔄 Data Freshness

Issue: Search results are cached for 24 hours, so recent certification changes may not appear immediately.

💡 Current Status:
  • 24-hour cache ensures fast responses but may delay new information
  • Cache automatically expires and refreshes after 24 hours
  • Background GPT analysis continues to improve cached data

🌍 Language Detection Limitations

Issue: Mixed-language company names or uncommon naming conventions may not be detected correctly, leading to suboptimal search results.

💡 Current Status:
  • Simple regex-based detection works for most cases
  • Users can try alternative name formats if initial search fails
  • We are working on improved multilingual name matching

🔍 Source Availability

Issue: Web scraping depends on the structure of target websites. If websites change their structure or block access, certain sources may temporarily fail.

💡 Current Status:
  • Multiple data sources ensure redundancy (if one fails, others continue)
  • Regular monitoring and updates of scraping selectors
  • Graceful degradation - partial results if some sources fail

📰 News Article Quality Dependency

Issue: The accuracy of GPT analysis depends on the quality and completeness of news article content. Poorly written or incomplete articles may yield inaccurate results.

💡 Current Status:
  • Top 3 news articles analyzed for cross-validation
  • Source attribution allows users to review original content
  • We prioritize articles from reputable news sources when available

💡 Continuous Improvement

We are actively working to address these limitations. The platform improves with each search through background AI analysis and cache updates. Future versions will include:

  • Expanded data source coverage
  • Improved AI accuracy through model refinement
  • Real-time data updates
  • Enhanced language detection and matching
  • User feedback mechanism for data corrections
